# ai-delivery

[**English version**](README.md)

*一组面向非技术人员的 AI 开发工具，让没有编程基础的人也能做出真实的软件项目。*

---

## 作者简介

我是一个做了 20 多年的互联网行业人力资源从业者，自己没有任何技术基础，所知都跟技术团队一起工作时耳濡目染的。

技术专业人员可以无视本 repo，但对技术小白的你，这里面的每一个字都是我能用最通俗的话写出来的。

---

## 这是什么

这是一组帮你用 AI 做开发的工具（skills）。

你不需要会写代码，不需要懂技术。只需要你知道自己想要什么——比如"我想做一个记账工具"或"我想做一个能自动整理文件的小工具"。

然后把这件事告诉 AI，AI 会帮你把事情做出来。

---

## 你能做什么

- 一个个人网站或博客
- 一个小型 Web 应用（如记账工具、习惯追踪器）
- 一个自动化处理重复任务的小工具
- 一个工作中用的 AI 助手
- 一个数据处理或报告生成工具

---

## 你需要什么

1. **知道自己想要什么** — 不需要技术细节，"我想做一个记账的网站"就够了
2. **Claude Code** — 一个免费的 AI 工具，从 [code.claude.com](https://code.claude.com) 下载
3. **一台电脑** — Mac、Windows 或 Linux，就这些

---

## 怎么用

**第一步 — 下载 skill**

```bash
git clone https://github.com/maxilen/ai-delivery.git
```

把 skill 文件放到你的项目文件夹里：

```bash
# 如果你已经有项目文件夹了：
cp ai-delivery/.claude/skills/dev-delivery.md /你的项目/.claude/skills/

# 如果是从零开始，直接把 skills 文件夹复制进去：
cp -r ai-delivery/.claude /你的新项目/
```

**第二步 — 告诉 Claude Code 你想做什么**

在项目文件夹里打开 Claude Code，说：

```
用 dev-delivery skill 帮我开发 [你的想法]
```

Claude Code 会自动加载 skill，理解你的项目，制定开发计划，执行开发，测试，最后交付结果。

---

## 工作流程是什么样的

skill 会把所有工作拆成小步骤：

1. **理解你的项目** — Claude 读你的文件，了解你目前有什么
2. **制定计划** — 把工作拆到最小颗粒度，安排好每一步怎么测试
3. **一步一步开发** — 按计划执行，每一步都附带测试
4. **集成测试** — 多个模块拼在一起时，做回归测试
5. **交付或暂停决策** — 如果需要你做决定，会清楚列出选项等你选

你随时可以停下来想一想、改变主意，然后告诉 Claude 继续。skill 会记住做到哪了。

---

## 我需要懂技术吗

不需要。你只需要描述你想要什么。Claude Code 加载这个 skill 后会自动处理所有技术工作。

但你需要做决定。比如：
- "我更喜欢方案 A" — 或者 —
- "这个地方感觉不对，能改吗？"

这些决定是创作过程的一部分，不是技术过程。

---

## 中途可以改需求吗

可以。这是正常且预期的。

你随时可以说：
- "我想加一个功能：……"
- "其实可以把 X 改成 Y 吗？"
- "现在的结果不错，但需要它能做 Z"

skill 会记录项目状态，从你停下的地方继续。

---

## 如果做到一半卡住了怎么办

skill 会用大白话告诉你它需要什么。比如：

```
我需要你对两件事做决定：

1. 登录用邮箱还是手机号？
   - 方案 A：邮箱（更专业）
   - 方案 B：手机号（用户更快）

2. 数据存在本地还是云端？
   - 方案 A：本地（免费、隐私，但只有在这台电脑上能用）
   - 方案 B：云端（在哪都能访问，但需要注册账号）
```

你只要选一个，work 就继续。
